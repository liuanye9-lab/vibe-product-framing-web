## Bug 修复 #3：模型输出截断导致 JSON 不完整（commit `46a5bd1`）

**问题**（用户 Vercel 线上报错）：
"模型返回格式错误，未找到有效 JSON。原始返回（前200字）：`{\"technical\":{\"architecture\":{\"value\":\"单页HTML+Tailwind...\",\"reason\":\"...\"},...},\"whyNoBackend\":{`"

**根因**：
1. **suggestStage 对所有阶段统一使用 maxTokens=850**，但 technical 阶段有 15+ 字段，远超预算
2. 模型输出到一半被 max_tokens 截断 → JSON 花括号不闭合 → parse 失败
3. 无截断检测/恢复机制——即使只差几个字符也无法挽救

**修复**：
- **STAGE_TOKEN_BUDGET 按阶段分配 token 预算**：discovery:900, product:1000, business:1200, **technical:2400**, mvp:1400, blindSpot:1300
- suggestIdeaDiagnosis: 850→2000（3 阶段合并输出）
- **新增 attemptTruncatedJsonRecovery()**：逐字符扫描器，自动补全未闭合的字符串/数组/对象括号
- **新增 detectTruncationHint()**：截断启发式检测，错误消息中报告具体未闭合数量
- tryRepairAndParse 管线从 6 策略扩展到 **8 策略**（新增 truncated recovery + balanced+truncated recovery）
- 范围：仅 evaluate.ts（+170/-3 行），零 UI 变更
